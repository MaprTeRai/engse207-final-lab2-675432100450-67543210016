## ขอบเขตงานที่รับผิดชอบ
ในโครงการ Microservices Architecture นี้ ข้าพเจ้ารับผิดชอบในส่วนของการพัฒนา service หลักของระบบ รวมถึงการตั้งค่าโครงสร้างของระบบ backend และการเชื่อมต่อระหว่าง services ต่าง ๆ โดยมีรายละเอียดดังนี้

- พัฒนา Auth Service สำหรับการจัดการระบบ login และ JWT authentication
- พัฒนา Task Service สำหรับการจัดการข้อมูล task ของผู้ใช้งาน
- ออกแบบและพัฒนา route และ middleware ภายใน services
- จัดการการเชื่อมต่อกับฐานข้อมูลของระบบ
- ตั้งค่า Docker และ Docker Compose สำหรับการรัน services หลายตัวพร้อมกัน
- ตั้งค่า Nginx Reverse Proxy เพื่อเป็น Gateway ของระบบ

## สิ่งที่ได้ดำเนินการด้วยตนเอง
ข้าพเจ้าได้ดำเนินการพัฒนาและตั้งค่าระบบในหลายส่วนเพื่อให้ services ต่าง ๆ สามารถทำงานร่วมกันได้อย่างถูกต้อง โดยมีรายละเอียดดังนี้

- พัฒนา API ของ Auth Service สำหรับการ login และสร้าง JWT Token
- เขียน middleware สำหรับตรวจสอบ JWT Token ก่อนเข้าถึง API ของระบบ
- พัฒนา API ของ Task Service สำหรับการสร้าง แสดง แก้ไข และลบข้อมูล task
- ออกแบบโครงสร้าง route ของแต่ละ service เพื่อให้การจัดการ API เป็นระบบมากขึ้น
- ตั้งค่าการเชื่อมต่อกับฐานข้อมูลเพื่อใช้เก็บข้อมูลผู้ใช้และข้อมูล task
- ตั้งค่า Dockerfile สำหรับแต่ละ service เพื่อให้สามารถรันใน container ได้
- สร้างไฟล์ Docker Compose เพื่อจัดการ container หลายตัวในระบบเดียว
- ตั้งค่า Nginx Reverse Proxy เพื่อทำหน้าที่เป็น Gateway สำหรับรับ request จาก client และส่งต่อไปยัง service ภายในระบบ

## ปัญหาที่พบและวิธีการแก้ไข
ระหว่างการพัฒนาระบบพบปัญหาหลายประเด็น ดังนี้

1. ปัญหาการสื่อสารระหว่าง Services
ในช่วงแรกของการพัฒนาระบบพบว่าบาง service ไม่สามารถเรียกใช้งาน service อื่นได้ เนื่องจากการตั้งค่า network ภายใน Docker ไม่ถูกต้อง
วิธีการแก้ไขคือการกำหนด network ภายใน Docker Compose ให้ container ทุกตัวอยู่ใน network เดียวกัน และใช้ชื่อ service ในการอ้างอิงระหว่าง container แทนการใช้ localhost

2. ปัญหาเกี่ยวกับ JWT Authentication

## สิ่งที่ได้เรียนรู้จากงานนี้
จากการทำโครงการนี้ทำให้ข้าพเจ้าได้เรียนรู้เกี่ยวกับแนวคิดของ Microservices Architecture ซึ่งเป็นการแยกระบบออกเป็น service ย่อยหลายตัว โดยแต่ละ service มีหน้าที่รับผิดชอบเฉพาะด้าน

นอกจากนี้ยังได้เรียนรู้เกี่ยวกับการออกแบบ API และการใช้ JWT Token สำหรับการยืนยันตัวตนของผู้ใช้งาน รวมถึงการใช้ middleware เพื่อตรวจสอบสิทธิ์ก่อนเข้าถึง resource ต่าง ๆ

อีกสิ่งหนึ่งที่ได้เรียนรู้คือการใช้ Docker และ Docker Compose ในการจัดการ container หลายตัว ทำให้สามารถจำลองสภาพแวดล้อมของระบบที่มีหลาย services ได้สะดวกมากขึ้น

นอกจากนี้ยังได้เรียนรู้การใช้ Nginx Reverse Proxy เพื่อทำหน้าที่เป็น Gateway ของระบบ และช่วยจัดการ routing ของ request ไปยัง service ที่เหมาะสม
