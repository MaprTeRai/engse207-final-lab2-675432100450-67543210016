## สิ่งที่ได้ดำเนินการด้วยตนเอง
ข้าพเจ้าได้ดำเนินการทดสอบและตรวจสอบระบบในหลายส่วน เพื่อให้แน่ใจว่า services ต่าง ๆ สามารถทำงานร่วมกันได้อย่างถูกต้อง โดยมีรายละเอียดดังนี้

- ทดสอบ API ของ Auth Service ผ่าน Postman เพื่อทำการ login และรับ JWT Token
- ทดสอบการเข้าถึง API ของ Task Service โดยใช้ JWT Token เพื่อยืนยันตัวตน
- ทดสอบการสร้าง task ใหม่ (Create Task) ผ่าน API
- ทดสอบการดึงข้อมูล task ทั้งหมด (Get Tasks) จากระบบ
- ทดสอบการแก้ไขข้อมูล task (Update Task) และการลบ task (Delete Task)
- ทดสอบกรณีที่ไม่มี JWT Token เพื่อให้ระบบตอบกลับด้วยสถานะ 401 Unauthorized
- ตรวจสอบการทำงานของ Nginx Reverse Proxy ที่ทำหน้าที่เป็น Gateway ของระบบ
- ตรวจสอบการเข้าถึงระบบผ่าน HTTPS และการเชื่อมต่อระหว่าง container ภายใน Docker

## ปัญหาที่พบและวิธีการแก้ไข
ระหว่างการทดสอบระบบพบปัญหาหลายประเด็น ดังนี้

1. ปัญหา JWT Token ไม่ถูกต้อง
ในช่วงแรกของการทดสอบ API พบว่าเมื่อเรียกใช้งาน API ของ Task Service ระบบตอบกลับด้วยข้อความ "Unauthorized: jwt malformed" ซึ่งเกิดจากการที่ Token ที่ส่งไปใน Header ไม่ถูกต้อง หรือไม่ได้ส่งในรูปแบบที่กำหนด
วิธีการแก้ไขคือการตรวจสอบรูปแบบของ Authorization Header ให้ถูกต้อง โดยต้องส่งในรูปแบบ
Authorization: Bearer
หลังจากแก้ไขรูปแบบการส่ง Token แล้ว ระบบสามารถตรวจสอบสิทธิ์และเข้าถึง API ได้ตามปกติ

2. ปัญหาการเข้าถึง API ผ่าน Nginx
ในบางกรณีเมื่อเรียกใช้งาน API ผ่าน Nginx Gateway ระบบแสดงผลเป็น 404 Not Found ซึ่งเกิดจากการตั้งค่า routing ของ Nginx ที่ไม่ตรงกับ path ของ service
วิธีการแก้ไขคือการตรวจสอบไฟล์ configuration ของ Nginx และปรับเส้นทาง proxy_pass ให้ตรงกับ service ภายใน Docker network หลังจากแก้ไขแล้วสามารถเรียก API ผ่าน Gateway ได้ตามปกติ

## สิ่งที่ได้เรียนรู้จากงานนี้
จากการทำโปรเจกต์นี้ทำให้ข้าพเจ้าได้เรียนรู้แนวคิดและเทคโนโลยีที่เกี่ยวข้องกับ Microservices Architecture มากขึ้น โดยเฉพาะในด้านการแยก services ออกจากกันและให้แต่ละ service รับผิดชอบหน้าที่เฉพาะ

นอกจากนี้ยังได้เรียนรู้เกี่ยวกับการทำงานของ JWT Authentication ซึ่งใช้ในการยืนยันตัวตนของผู้ใช้งานก่อนเข้าถึง API ต่าง ๆ รวมถึงการทำงานของ Reverse Proxy ผ่าน Nginx ที่ทำหน้าที่เป็น Gateway สำหรับจัดการ request จาก client ไปยัง service ต่าง ๆ ภายในระบบ

อีกประเด็นหนึ่งที่ได้เรียนรู้คือการใช้ Docker และ Docker Compose เพื่อจัดการ container หลายตัวพร้อมกัน ทำให้สามารถจำลองระบบที่มีหลาย services ได้ง่ายขึ้น และช่วยให้การ deploy ระบบเป็นไปอย่างมีระเบียบมากขึ้น

นอกจากนี้ยังได้เรียนรู้การทดสอบ API ด้วย Postman และการตรวจสอบการทำงานของระบบแบบ End-to-End เพื่อให้มั่นใจว่าทุก service สามารถทำงานร่วมกันได้อย่างถูกต้อง
