## การแบ่งงานหลัก

### สมาชิกคนที่ 1: นายวีราวัชร์ สุประดิษฐ์พงศ์
รับผิดชอบงานหลักดังต่อไปนี้
- การพัฒนา Microservices และการตั้งค่า Backend Infrastructure ของระบบ โดยมีรายละเอียดดังนี้

# การพัฒนา Service หลักของระบบ
- ออกแบบและพัฒนา Auth Service
- พัฒนา User Service
- พัฒนา Task Service
- ออกแบบโครงสร้าง API สำหรับแต่ละ service

# การออกแบบระบบฐานข้อมูล
- ออกแบบ schema ของฐานข้อมูลสำหรับแต่ละ service
- แยกฐานข้อมูลตามแนวคิด Database per Service
- จัดการการเชื่อมต่อระหว่าง service กับ PostgreSQL

# การพัฒนา Authentication System
- พัฒนา Register API
- พัฒนา Login API
- สร้างและตรวจสอบ JWT Token
- จัดการ middleware สำหรับตรวจสอบสิทธิ์การเข้าถึง API

# การพัฒนา API และ Business Logic
- พัฒนา API สำหรับจัดการ Task (CRUD)
- พัฒนา API สำหรับจัดการ User Profile
- พัฒนา API สำหรับตรวจสอบสิทธิ์ผู้ใช้งาน

# การตั้งค่า Environment และ Container
- สร้าง Dockerfile สำหรับแต่ละ service
- ตั้งค่า Docker Compose
- จัดการ Environment Variables ของระบบ
- แก้ไขปัญหาการเชื่อมต่อระหว่าง container

# การ Deploy ระบบ
- Deploy services ขึ้น Railway Cloud Platform
- ตั้งค่า environment variables สำหรับ production
- เชื่อมต่อ service กับ database บน cloud

### สมาชิกคนที่ 2: [นายชยานันท์   เพชรรักษ์]
รับผิดชอบงานหลักดังต่อไปนี้
- รับผิดชอบงานหลักในส่วนของ การทดสอบระบบ การตรวจสอบการทำงาน และการจัดทำเอกสารประกอบโครงงาน โดยมีรายละเอียดดังนี้

# การทดสอบระบบ (System Testing)
- ทดสอบ API ของทุก service ผ่าน Postman
- ตรวจสอบการทำงานของระบบ Authentication
- ทดสอบการใช้งาน JWT Token ระหว่าง services
- ทดสอบการเข้าถึง API ที่ต้องมีสิทธิ์ผู้ใช้
- ตรวจสอบการตอบสนองของระบบในกรณี error

# การทดสอบ End-to-End
- ทดสอบ flow การใช้งานตั้งแต่
- Register
- Login
- Create Task
- Update Task
- Delete Task
- ตรวจสอบความถูกต้องของข้อมูลที่จัดเก็บในฐานข้อมูล
- ตรวจสอบการทำงานของระบบหลัง deploy

# การทดสอบความปลอดภัยของ API
- ทดสอบกรณี ไม่มี JWT Token
- ทดสอบกรณี JWT ไม่ถูกต้อง
- ตรวจสอบ response code ของ API เช่น
- 200 OK
- 401 Unauthorized
- 403 Forbidden

สมาชิกทั้งสองคนได้ร่วมกันดำเนินการในส่วนสำคัญของโครงงานดังต่อไปนี้

- จัดเตรียม screenshots สำหรับการส่งงาน

## เหตุผลในการแบ่งงาน
การแบ่งงานของกลุ่มนี้ใช้แนวทาง แบ่งตามบทบาทของการพัฒนาระบบ (Role-based responsibility)

สมาชิกคนที่ 1 รับผิดชอบด้าน การพัฒนา backend และการตั้งค่าระบบ
สมาชิกคนที่ 2 รับผิดชอบด้าน การทดสอบระบบ การตรวจสอบการทำงาน และการจัดทำเอกสาร

แนวทางนี้ช่วยให้การทำงานมีความชัดเจน และสามารถพัฒนาระบบได้อย่างมีประสิทธิภาพ

## สรุปการเชื่อมโยงงานของสมาชิก
การทำงานของสมาชิกทั้งสองคนมีการเชื่อมโยงกันดังนี้

- 1. สมาชิกคนที่ 1 พัฒนา services และ API ของระบบ

- 2. สมาชิกคนที่ 2 ทำการทดสอบ API และตรวจสอบผลลัพธ์

- 3. เมื่อพบปัญหาในการทดสอบ สมาชิกคนที่ 2 จะรายงานเพื่อให้สมาชิกคนที่ 1 แก้ไข

- 4. หลังจากระบบทำงานถูกต้องแล้ว สมาชิกคนที่ 2 จะจัดทำเอกสารและรวบรวม screenshots เพื่อใช้ในการส่งงาน

รูปแบบการทำงานนี้ช่วยให้ระบบมีการตรวจสอบคุณภาพก่อนส่ง และทำให้โครงงานมีความสมบูรณ์มากขึ้น
